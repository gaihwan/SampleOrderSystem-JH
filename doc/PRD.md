# PRD — 반도체 시료 생산주문 관리 시스템 (SampleOrderSystem)

## 1. 배경 및 문제 정의

### 회사 개요

**S-Semi** 는 반도체 시료(Sample) 전문 제조사로, Fabless 업체 및 연구기관에 소량의 반도체 시료를 공급한다.

### 문제 상황

현재 시료 생산 주문 프로세스는 영업팀, 생산팀, 고객 간 커뮤니케이션이 체계적으로 관리되지 않아 다음과 같은 문제가 발생하고 있다.

> **"언제 납품이 가능한가요?"**
> **"잘 모르겠습니다."**
> **"왜 주문했는데 생산이 안 되고 있나요?"**

- 주문 상태를 실시간으로 추적할 수 없다
- 생산 가능 여부와 납기 예측이 불가능하다
- 주문 거절/확정/생산 상태가 부서 간에 공유되지 않는다

### 해결 방향

**S-Semi** 는 위 문제를 해결하기 위해 **시료 생산주문 관리 시스템**을 도입한다.  
주문의 전체 라이프사이클을 단일 시스템에서 추적하고, 생산 가능 수량과 납기를 자동으로 계산한다.

---

## 2. 목표 (Goals)

| # | 목표 | 측정 기준 |
|---|------|---------|
| G1 | 주문 상태를 실시간으로 조회할 수 있다 | 모든 주문의 현재 상태가 화면에 표시됨 |
| G2 | 생산 가능 수량과 납기를 자동 계산한다 | 수율 기반 배치 수 계산 공식 적용 |
| G3 | 주문 라이프사이클 전체를 단일 시스템에서 관리한다 | RESERVED → RELEASE 전 상태 추적 가능 |
| G4 | 부서 간 커뮤니케이션 오류를 최소화한다 | 상태 변경 시 즉시 반영 |

| 역할 | 설명 |
|------|------|
| **영업 담당자** | 주문 등록, 주문 취소 요청 |
| **생산 관리자** | 주문 확정(CONFIRMED), 생산 시작(PRODUCING), 출하(RELEASE), 주문 거절(REJECTED) |

---

## 5. 주문 상태 라이프사이클

```
[주문 등록]
     │
     ▼
  RESERVED ──────────────────────────► REJECTED
     │                                   (거절)
     │ 생산 확정
     ▼
  CONFIRMED ─────────────────────────► REJECTED
     │                                   (거절)
     │ 생산 시작
     ▼
  PRODUCING
     │
     │ 출하
     ▼
  RELEASE
```

| 상태 | 설명 |
|------|------|
| **RESERVED** | 주문 접수 완료, 확정 대기 중 |
| **REJECTED** | 주문 거절 (RESERVED 또는 CONFIRMED 상태에서 가능) |
| **CONFIRMED** | 생산 확정, 생산 시작 대기 |
| **PRODUCING** | 생산 진행 중 |
| **RELEASE** | 출하 완료 |

## 6. 핵심 기능 요약

### F1. 주문 목록 조회
- 전체 주문을 상태, 시료 정보, 수량, 납기와 함께 표시
- 상태별 필터링 지원

### F2. 시료 주문 생성
- 시료 ID, 수량, 납기일 입력
- 수율(Yield Rate)을 고려한 생산 배치 자동 계산
- 생성 직후 RESERVED 상태

### F3. 주문 확정 / 거절
- 생산 관리자가 RESERVED 주문을 CONFIRMED 또는 REJECTED 처리
- 확정 시 납기 및 생산 배치 수 표시

### F4. 생산 시작
- CONFIRMED 주문을 PRODUCING으로 전환
- 생산 중인 주문은 수정 불가

### F5. 출하 처리
- PRODUCING 주문을 RELEASE로 전환

### F6. 주문 취소 / 강제 거절
- RESERVED, CONFIRMED, PRODUCING, RELEASE 상태의 주문을 REJECTED 처리 가능

---

## 7. 핵심 비즈니스 규칙

1. **수율 계산**: 실제 생산 가능 수량 = 배치 수 × 배치당 정상 수량 × 수율(0.9)
2. **필요 배치 수**: `ceil(주문수량 / (배치당수량 × 0.9))`
3. **납기 계산**: 배치 수 × 배치당 생산 기간
4. **생산 순서**: FIFO (먼저 확정된 주문이 먼저 생산됨)
5. **상태 전이 제약**: 각 상태에서 허용된 전이만 가능 (라이프사이클 참조)

## 9. 성공 기준

- 전체 주문 라이프사이클(RESERVED → RELEASE)이 시스템 내에서 완결된다
- 수율 기반 납기 계산이 자동으로 이루어진다
- 모든 기능이 GoogleTest로 검증된 단위 테스트를 가진다
- 콘솔 UI에서 주문 목록과 상태를 직관적으로 확인할 수 있다
